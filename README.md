# Anthology-Manuscript-Manager
Google Colabを用いたアンソロジー原稿管理・プレビュー生成プログラムです。
使用例は[こちら](<https://drive.google.com/drive/folders/1QKdZM1k_vf9IBfG8TXsrDfER6ttp1Y3V?usp=drive_link>)です

複数の執筆者から提出された原稿を一括で確認・整理し、ノンブル付き原稿、見開きプレビュー、原稿チェック結果、入稿用PDFを自動作成します。原稿管理に伴う作業や確認待ちを減らし、主催者と執筆者の双方が原稿制作に集中できる環境を目指しています。

## 主な機能

- Google Drive上の執筆者別フォルダからPNG原稿を取得
- Googleスプレッドシートから執筆者名、参加形態、予定ページ数、掲載順を取得
- 原稿の画像サイズを自動確認
- モノクロ本の場合、カラーが含まれていないかを自動確認
- 不備のあるページを白紙に置き換え、ページ順を維持したまま出力
- 指定した原稿へのノンブル付与
- 原稿を全体の掲載順に並べ替え、3桁の連番で保存
- 執筆者ごとの原稿チェック結果をCSVで出力
- 執筆者別・主催者向けの見開きプレビューを生成
- ページ数に応じた白紙ページの補完
- 目次画像の生成（任意）
- 全原稿をまとめたPDFの生成

## 動作環境

- Google Colab
- Google Drive
- Googleスプレッドシート
- Python 3（Google Colabの実行環境）

使用する主なライブラリは次のとおりです。

- Pillow

- gspread
- natsort
- img2pdf

`img2pdf`はノートブック内でインストールされます。そのほかはGoogle Colabの標準環境で利用することを想定しています。

## 事前準備

### 1. Google Driveにフォルダを作成する

次の構成を基本として、Google Drive上にフォルダとファイルを用意してください。

```text
任意のプロジェクトフォルダ/
├── 原稿/
│   ├── 全ての原稿/
│   ├── その他/
│   │   └── nombre_image/
│   │       ├── number_0.png
│   │       ├── number_1.png
│   │       ├── ...
│   │       └── number_9.png
│   ├── 目次/
│   │   └── 素材/
│   ├── 執筆者A様/
│   │   ├── 原稿_001.png
│   │   └── 原稿n_002.png
│   └── 執筆者B様/
│       └── 原稿_001.png
└── 原稿管理.ipynb
```

`原稿/全ての原稿`には、処理後の原稿と主催者向けプレビューが出力されます。
執筆者A様に事前に原稿提出場所として`執筆者A様`のリンクを共有し、管理者権限を付与してください。

### 2. 執筆者名簿を作成する

Googleスプレッドシートに、次の名前のファイルとシートを作成します。

- スプレッドシート名：`執筆者名簿`
- シート名：`名簿`

1行目を見出しとし、2行目以降に次の順番で情報を入力してください。
ここでの並び順が掲載順となります。掲載順を変更したい場合は、このファイルを書き換えてください。

| 列 | 内容 | 入力例 |
|---|---|---|
| A | 管理用番号 | `1` |
| B | ペンネーム | `執筆者A` |
| C | 参加形態 | `漫画` |
| D | 予定ページ数 | `8` |

現状では、執筆者の参加形態は`イラスト`、`漫画`、`小説`のみとし、`原稿/{ペンネーム}様`が提出フォルダとして参照されます。
目次や奥付などの、それ以外の参加形態は、`原稿/{ペンネーム}`が参照されます。

### 3. 原稿ファイル名を整える

原稿はPNG形式で提出してください。ノートブックが認識する主なファイル名は次のとおりです。

```text
原稿_001.png      # ノンブルを付けない
原稿n_001.png     # ノンブルを付ける
原稿_001(1).png   # Google Driveで重複時の番号が付いたもの
原稿n_001(1).png  # ノンブルあり・重複番号あり
```

ファイル名の数字は、その執筆者の原稿内でのページ番号として扱われます。`n`を含む原稿には、アンソロジー全体でのページ番号がノンブルとして付与されます。規定外のファイル名は原稿として処理されず、チェック結果CSVの末尾に記録されます。

## 設定

ノートブックの「設定」セルを、制作する本の仕様に合わせて変更してください。

```python
# Google Drive上のプロジェクトフォルダ
folder_path = "/content/drive/MyDrive/任意のフォルダ"

# 入稿原稿の画像サイズ（横px, 縦px）
manuscript_size = (3638, 5102)

# 仕上がりサイズ（横px, 縦px）
book_size = (3496, 4961)

# True：モノクロ本、False：フルカラー本
color_type = True

# 解像度
dpi = 600

# "右綴じ" または "左綴じ"
toji = "右綴じ"
```

### 目次の設定

```python
# 目次を生成するか
mokuji_make = True

# 目次素材を配置し始める座標
mokuji_box_position = (330, 613)
```

目次機能は、使用するデザインに合わせて関数内の座標や素材を調整する必要があります。使用しない場合は`mokuji_make = False`にしてください。

### ノンブルの設定

```python
# ノンブル機能を使用するか
has_nombre_book = True

nombre_setting = [
    1.0,                         # ノンブル画像の倍率
    ("外", "下"),               # 横位置、縦位置
    (170, 170),                  # 原稿端からの余白
    True,                        # 背景の明暗に応じて色を変えるか
    [(0, 0, 0), (255, 255, 255)] # 明るい背景用、暗い背景用の色
]
```

ノンブルを使用する場合は、`原稿/その他/nombre_image`に`number_0.png`から`number_9.png`までの透過PNG画像を配置してください。必要な画像を読み込めない場合、ノンブル機能は自動的に無効になります。

## 使用方法

1. `原稿管理.ipynb`をGoogle Colabで開きます。
2. 「設定」セルを編集します。
3. 「利用準備」セルを実行し、Google DriveのマウントとGoogleアカウントの認証を行います。
4. クラス・関数を定義する各セルを上から順番に実行します。
5. 最後の「実行」セルを実行します。
6. Google Drive上に生成された原稿、CSV、プレビュー、PDFを確認します。

Google Driveとスプレッドシートへのアクセス許可を求められた場合は、原稿および名簿を保存しているGoogleアカウントで認証してください。

## 出力内容

### 全体原稿

処理後の原稿は、次のフォルダにアンソロジー全体のページ番号で保存されます。

```text
原稿/全ての原稿/001.png
原稿/全ての原稿/002.png
原稿/全ての原稿/003.png
...
```

画像サイズまたはカラー設定に不備があるページは、原稿の代わりに白紙画像が保存されます。

### 原稿チェック結果

各執筆者の提出フォルダに、次のCSVファイルが出力されます。

```text
{ペンネーム}様_原稿チェック結果.csv
```

CSVには、予定ページ数との不一致、未提出ページ、画像サイズ、カラー混入、認識されなかったファイル名などが記録されます。

| 表示 | 意味 |
|---|---|
| `[PAGE OVER]` | 予定ページ数を超える原稿が提出されています |
| `[PAGE ODD]` | ページ配置を揃えるため、末尾に白紙ページを追加しました |
| `[COLOR NG]` | モノクロ本の原稿にカラーが検出されました |
| `[SIZE NG]` | 原稿の画像サイズが設定値と一致していません |

CSVの結果は自動判定です。最終的な入稿前には、**必ず主催者が原稿を目視で確認してください。**

### プレビュー

- 執筆者向け：`原稿/{ペンネーム}様/プレビュー/`
- 主催者向け：`原稿/全ての原稿/プレビュー/`

原稿を仕上がりサイズで中央からトリミングし、綴じ方向に合わせた見開き画像を出力します。
執筆者向けは、見開き時左右で違う執筆者の作品が掲載される場合に、執筆者が自身以外の原稿を白いページにして出力します。
主催者向けは、白抜きなどはなく、すべてのページの見開きを出力します。

### PDF

すべての処理後原稿をページ順にまとめた`提出.pdf`が、プロジェクトフォルダ直下に出力されます。印刷所がPNGでの入稿を指定している場合は、PNGファイルを使用してください。

## 個別に再チェックする方法

ノートブック末尾の個別チェック用セルで、対象者の名前を指定します。

```python
check_name = "花子"
```

名簿の読み込みと`authors`の作成後にこのセルを実行すると、指定した執筆者の原稿を再確認できます。

## 注意事項・既知の制限

- 実行前に、原稿ファイルと出力先フォルダのバックアップを取ってください。
- 画像サイズは縦横とも設定値と1pxでも異なる場合、`[SIZE NG]`になります。
- モノクロ判定は画像のRGB各チャンネルの差をもとに行います。印刷所のカラープロファイルや入稿規定を保証するものではありません。
- 不備のあるページは全体原稿上で白紙に置き換えられます。CSVを確認し、修正後に再実行してください。
- 目次生成機能は特定のデザインと素材配置を前提としているため、配布先の企画に合わせたコード修正が必要です。
- PDF生成後も、ページ順、左右の配置、ノンブル、塗り足し、カラーモードなどを目視で確認してください。
- Google Colabや各ライブラリの仕様変更により、将来動作しなくなる可能性があります。

## トラブルシューティング

### `spleadsheet 執筆者名簿が見つかりません`と表示される

スプレッドシート名が`執筆者名簿`、シート名が`名簿`になっているか確認してください。また、Colabで認証したGoogleアカウントに閲覧権限があるか確認してください。

### 執筆者のフォルダが読み込まれない

名簿のペンネームとフォルダ名が一致しているか確認してください。`イラスト`、`漫画`、`小説`の参加者は、フォルダ名の末尾に`様`が必要です。

### 原稿が認識されない

ファイル形式がPNGであること、ファイル名が`原稿_001.png`または`原稿n_001.png`などの形式になっていることを確認してください。認識されなかったファイルは、執筆者フォルダ内のチェック結果CSVにも表示されます。

### ノンブルが付かない

次の項目を確認してください。

- `has_nombre_book = True`になっている
- ファイル名に`n`が含まれている
- `number_0.png`から`number_9.png`までの素材が所定のフォルダにある
- ノンブル素材が透過PNGになっている

ノンブルのための数字画像がうまく読み込めていない場合、セルの出力にその旨が出力されますので、うまくいかない場合はそちらもご確認ください。

### 目次が作成されない

`mokuji_make = True`になっていることと、`原稿/目次/素材`に必要な土台・数字・執筆者別素材があることを確認してください。目次の素材名や配置は企画ごとに調整が必要です。
素材画像がうまく読み込めていない場合、セルの出力にその旨が出力されますので、うまくいかない場合はそちらもご確認ください。
